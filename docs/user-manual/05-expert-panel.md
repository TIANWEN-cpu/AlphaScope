# 专家团使用

专家团是 AI-FINANCE 的核心功能，通过多位专家协作提供全面的投资分析。

---

| 专家 | 角色 | 投资风格 | 关注维度 |

## 使用专家团

### 通过界面

1. 点击左侧 **专家团**
2. 选择 **股票合伙人团队**
3. 输入股票代码（如 `600519`）
4. 点击 **启动分析**
5. 等待分析完成（约 30-60 秒）

### 通过 API

```bash
curl -X POST http://localhost:8000/api/analysis/run \
  -H "Content-Type: application/json" \
  -d '{
    "stock_symbol": "600519",
    "stock_name": "贵州茅台",
    "mode": "deep"
  }'
```

## 分析模式

### 标准模式（Standard）

- 3 个 Agent：基本面、技术面、情绪面
- 使用 DeepSeek 模型
- 快速、低成本
- 适合日常快速查看

### 深入模式（Deep）

- 5 个 Agent + Critic + 主席
- 使用 5 家厂商异构模型
- 全面、高质量
- 适合重要决策

### 自动模式（Auto）

- 先用标准模式预筛
- 信号不明确时自动升级到深入模式
- 平衡成本与质量

## 自定义专家团

### 通过界面

1. 点击左侧 **专家团编辑器**
2. 点击 **新建专家团**
3. 填写专家团信息：
   - 名称
   - 描述
   - 输出格式
4. 添加专家成员：
   - 选择专家角色
   - 配置人设（System Prompt）
   - 选择模型
   - 设置止损风格
5. 保存专家团

### 通过配置文件

编辑 `config/experts.yaml`：

```yaml
teams:
  - id: my-team
    displayName:
      zh: 我的专家团
    description:
      zh: 自定义专家团
    members:
      - id: my-expert
        displayName:
          zh: 我的专家
        provider: deepseek
        model: deepseek-chat
        promptFile: prompts/experts/my-expert.md
```

## 专家输出格式

每位专家输出结构化 JSON：

```json
{
  "view": "核心观点（1-2 句）",
  "evidence": [
    {
      "type": "fundamental",
      "claim": "具体依据（含数字）",
      "data_date": "2026-05-21"
    }
  ],
  "action": "买入|观望|减持|卖出",
  "position": 30,
  "stop_loss": 1500.0,
  "invalid_if": "失效条件",
  "risks": ["风险点1", "风险点2"]
}
```

## 审稿员（Critic）

深入模式下，审稿员会评估每位专家的输出：

| 维度 | 说明 |
|------|------|
| 证据质量 | 证据是否充分、可靠 |
| 逻辑一致性 | 推理是否自洽 |
| 矛盾检测 | 是否与其他专家矛盾 |
| 过度自信 | 是否高估置信度 |

审稿评分：
- 80+ 高质量
- 60-79 可参考
- 40-59 待商榷
- <40 存疑

---

## 主席总结

深入模式下，主席会综合所有专家观点：

1. 统计投票（买入/卖出/观望）
2. 计算平均置信度
3. 综合分歧点
4. 输出最终决策

## 常见问题

### Q: 分析需要多长时间？

- 标准模式：10-20 秒
- 深入模式：30-60 秒
- 自动模式：10-60 秒（视情况）

### Q: 可以同时运行多个分析吗？

可以，但建议控制并发数量，避免 API 限流。

### Q: 如何提高分析质量？

1. 配置更多 LLM 提供商（异构模型）
2. 使用深入模式
3. 确保数据源可用
4. 自定义专家人设

### Q: 专家团配置保存在哪里？

- 界面配置：SQLite 数据库
- 文件配置：`config/experts.yaml`
