# AI-Finance v0.7 增量 PRD（简单版）

> 编写人：Alice（产品经理）  
> 版本：v0.7 增量需求  
> 适用范围：在 v0.6 基础上的功能增量，不重述已有能力

---

## 一、产品目标

v0.6 已具备「数据采集 + 5 Agent 分析报告 + 研究存档」的闭环，但用户仍处于**被动阅读**状态。v0.7 要解决三个痛点：**信息维度浅**（只有 K线/资金/资讯，缺财报/股东/同业）、**交互单向**（无法追问）、**视角同质**（5 Agent 都是分析师，缺立场鲜明的投顾）。

目标：让看板从「自动报告生成器」升级为「可交互的多视角投研工作台」——用户既能拿到全维度静态信息，也能就具体股票进行 AI 对话，并获取风格化策略建议。

## 二、用户故事

**US-1 财务追溯**  
- **Given** 用户已选中 600519  
- **When** 进入「💡基本面」Tab  
- **Then** 看到最近 4 期营收/净利/毛利率/ROE/资产负债率折线图，以及十大股东列表和机构持仓变动

**US-2 同业横评**  
- **Given** 用户已选中股票  
- **When** 滚动到「💡基本面」Tab 底部  
- **Then** 看到该股所属行业 5-8 家龙头的 PE/PB/ROE/营收增速/净利增速对比表，本股票高亮

**US-3 主力资金识别**  
- **Given** 用户已选中股票  
- **When** 进入「💰资金流向」Tab  
- **Then** 在原有总资金流下方看到超大/大/中/小单分布饼图与 5 日趋势，并附一行判定（主力洗盘 / 游资进场 / 散户接盘）

**US-4 AI 追问**  
- **Given** 用户在任意 Tab 浏览中，已选中股票  
- **When** 点开右下角悬浮的「🤖 AI 咨询」面板，输入「现在估值贵不贵？」  
- **Then** 系统自动注入当前股票上下文（代码/最新价/K线摘要/主力方向/最新 Agent 报告），DeepSeek 返回带数据依据的回答，可继续追问

**US-5 专家圆桌**  
- **Given** 用户已选中股票  
- **When** 进入「🎓专家团圆桌」Tab，点击「召开圆桌」  
- **Then** 5 位专家（巴菲特派/林奇派/缠论派/宏观官/风控官）并行输出三段式 JSON，UI 呈现 5 张卡片，底部统计「X买/Y观/Z减/W卖」，可一键导出 Markdown 纪要

## 三、需求池

### P0 — 必须实现

| 编号 | 需求 | 描述 | 验收标准 |
|---|---|---|---|
| P0-1 | 财务摘要 | 最近 4 期（年报或季报）核心指标：营收、净利、毛利率、ROE、资产负债率 | akshare 接口成功；折线图 + 同比柱图渲染正常；数据缺失时显示「该股暂无完整财报」 |
| P0-2 | 股东结构 | 十大股东 / 十大流通股东 / 机构持仓变动（进出） | 三表格在「💡基本面」Tab 分区展示；接口失败时占位提示 |
| P0-3 | 同业对比 | 自动识别股票所属行业，行业内 5-8 家龙头横向对比 | 表内 PE/PB/ROE/营收增速/净利增速；当前股票行高亮；行业识别失败降级为「申万一级行业」 |
| P0-4 | 资金流细化 | 超大单/大单/中单/小单分布饼图 + 5 日趋势柱图 | 「💰资金流向」Tab 内追加；附判定文字（规则：超大单净流入 > 5000 万 → 主力进场；反之主力出货） |
| P0-5 | 资讯时间轴 | 最近 10 条公告/快讯/研报合并按时间倒序 | 「📰资讯研报」Tab 顶部新增「时间轴 / 分类」视图切换器 |
| P0-6 | AI 咨询对话框 | 右下角悬浮按钮，点击展开侧滑面板；支持多轮对话 | 面板宽度 400px；消息持久化到 session_state；关闭再开历史保留 |
| P0-7 | 上下文自动注入 | 提问时拼接当前股票代码、最新价、近 5 日 K线摘要、主力资金方向、最新 Agent 报告摘要进 system prompt | 请求日志可见完整 context；无选中股票时禁用对话框并提示「请先选择股票」 |
| P0-8 | 模型可切换 | 默认 DeepSeek，配置层可切 Kimi/Claude/GPT | `.env` 通过 `AI_CHAT_PROVIDER` 切换；切换后下次提问即生效，无需重启 |
| P0-9 | 引导问题预设 | 4-6 个快捷按钮：估值分析、短期走势、风险提示、与同行对比、最新政策影响、机构评级 | 点击即填入输入框并自动发送 |
| P0-10 | 对话导出 | 一键导出当前会话为 Markdown | 文件名 `chat_{stock_code}_{timestamp}.md`，落盘到 `reports/chat_history/` |
| P0-11 | 专家团角色配置 | 5 位专家的 system prompt 与推荐模型映射写入 `config/experts.yaml` | 配置文件存在；每位专家独立人设、关注维度、止损偏好 |
| P0-12 | 专家团并行调用 | 5 位专家并行调用各自推荐 LLM，输出三段式 JSON | 单专家失败不阻塞其他（返回 error 状态卡片）；总耗时 ≤ 15 秒 |
| P0-13 | 专家团 UI | 5 卡片横排，可点击展开；底部统计「X买/Y观/Z减/W卖」 | 卡片正面显示 view + action + position；展开显示 evidence 列表与 stop_loss |
| P0-14 | 圆桌纪要导出 | 一键导出当次圆桌为 Markdown | 文件名 `roundtable_{stock_code}_{timestamp}.md`，落盘到 `reports/roundtables/` |

### 4.2 AI 咨询 — **推荐：右下角悬浮按钮 + 侧滑面板**

**三种方案对比：**

| 方案 | 优点 | 缺点 |
|---|---|---|
| 侧边栏常驻 | 醒目、操作快 | 占用横向空间，挤压 K线/图表 |
| 新 Tab | 不挤占横向空间 | Tab 已 7 个，再加更拥挤；离开 Tab 上下文断裂 |
| **浮动悬浮 + 侧滑（推荐）** | 不挤占任何 Tab，任何位置即开即用，关闭后保留历史 | 需自定义 CSS，Streamlit 原生支持有限 |

**推荐理由：** AI 咨询是**伴随式交互**——用户在 K线 Tab 看均线时随时想问「这意味着什么」，跳到独立 Tab 反而割裂上下文。  
**实现建议：** Streamlit 可用 `st.sidebar` + `st.expander` 折叠实现伪悬浮，或引入 `streamlit-chat` 组件。

## 五、待确认问题

1. **专家团模型映射在 LLM 不可用时如何降级？**  
   PRD 默认按需求文档把 5 位专家与 5 厂商一一绑定。但 v0.6 已有双层兜底机制——某厂商 API 不可用时，是按兜底逻辑切换备用模型（角色风格可能轻微漂移），还是当场返回「该专家暂不可用」？  
   **建议**：默认走兜底，但在卡片角标显示「(降级模型)」让用户知情。请主理人/用户确认。

2. **AI 咨询是否需要成本/频次上限？**  
   DeepSeek 单次便宜但高频追问累计仍有成本，且过长 context 影响响应速度。当前 P1-4 已规划「单会话 30 轮自动截断」，是否还需要：每股票每日提问次数上限（如 50 次）？跨会话总额度告警？  
   **建议**：v0.7 仅做 P1-4 的轮次截断，频次上限留待 v0.8 视实际用量再加。请确认。

3. **同业对比的「龙头」如何定义？**  
   akshare 的行业成分股可能数十至上百只，取「5-8 家龙头」需要规则：  
   - 选项 A：按行业内总市值降序前 8 家（PRD 默认方案）  
   - 选项 B：按所属指数（沪深 300 / 中证 500）成分筛选  
   - 选项 C：用户手动指定对比标的  
   **建议**：v0.7 用 A 方案（最简单稳定），B/C 留 P2。请确认。
